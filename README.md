# 프로젝트 소개

## 프로젝트 목적
1. **편리하고 쉬운 UI**: 앱에 익숙하지 않은 사용자도 쉽게 접근할 수 있는 UI 설계.
2. **플랫폼 통합화**: 다양한 OTT 플랫폼을 한 번에 검색하고 즐길 수 있음.
3. **강력한 추천 시스템**: 플랫폼별 시청 기록을 기반으로 맞춤 콘텐츠 추천.

## 문제 인식
- 다양한 OTT 플랫폼이 등장하면서 콘텐츠 추천의 중요성이 증가.
- 사용자가 원하는 콘텐츠를 쉽게 찾지 못하는 문제 발생.

## 해결 방안
- 시청 기록 DB와 추천 시스템을 활용한 플랫폼 통합 어플리케이션 개발.

## 기대 효과
- 완전한 플랫폼 통합을 통한 편리성 증대.

# 프로젝트 내용

## SW 구조 및 구성요소

Watch Me 앱은 OTT 플랫폼을 통합하고 개인화된 추천 시스템을 운영하는 것이 목표입니다.

### 주요 구성 요소
1. **프론트엔드**: 사용자 인터페이스(UI)와 UX 설계 (Flutter, Android Studio 사용)
2. **백엔드**: DB 관리 및 프론트엔드와 통신 (FastAPI 사용)
3. **추천 시스템**: 하이브리드 추천 적용 및 데이터 분석 (콘텐츠 기반 필터링 + 협업 필터링)

### 개발 초기 계획
- **백엔드**: 추천 시스템과 DB 및 프론트엔드 연결 방식 설계
- **프론트엔드**: 화면 전환도 설계 및 앱 UX 구성
- **추천 시스템**: 기존 논문과 baseline 코드 기반으로 최적화

## 주요 설계 고려사항 및 개발 항목

1. **OTT 데이터 수집**: API 활용 (TMDB, 키노라이츠 등)
2. **추천 시스템 구축**: 자연어 처리(NLP) 및 협업 필터링 적용
3. **검색 시스템 구현**: 키워드 검색 및 유사도 기반 검색 기능
4. **OTT 필터링 기능**: 사용자가 원하는 OTT 플랫폼 선택 가능
5. **유저 리뷰 및 평가 기능**: 평점 및 한줄평 남기기 기능 추가

## OTT 필터링 기능
- 특정 OTT 플랫폼만 필터링하여 원하는 콘텐츠 탐색 가능

## 개인별 추천 시스템

# 향후 개선 방안 및 보완 사항

## 서버 스펙 이슈 해결
- 현재 서버 성능 제한으로 인해 영화 데이터에만 적용됨
- 서버 업그레이드 후 드라마, 예능, 스포츠 콘텐츠까지 확장 계획

## DB 변경을 통한 쿼리 속도 개선
- Elasticsearch 사용 대신 NoSQL 저장소(Amazon DynamoDB) 도입 고려
- Bulk Query 지원 DB를 도입하여 성능 개선 예정

## 향후 계획
- 유저가 본 시청 목록 및 리뷰를 쉽게 관리할 수 있도록 마이페이지 개선
- SNS 기능 도입하여 유저 간 리뷰 공유 및 추천 기능 강화
- OTT 구독 분석 및 추천 기능 추가
- OTT 할인 정보 제공 및 수익성 향상 방안 연구
- 숏폼 콘텐츠 추천 기능 추가 계획
